Gaps Identified — Needs Resolution Before Build

1
No CRM / lead tracking — Between Lupe consolidating leads and Marcela responding, there's no persistent record. Leads can be lost.
→ Solution: Lupe creates one Asana task per inbound message in a permanent Leads project. Each task includes: source channel, category (project / speaking / collaboration / press), summary, date received, and status (new → Marcela reviewing → responded → qualified / discarded). Nothing falls through — every inbound has a record.
2
Client communication agent missing — Who sends updates, confirmations, and follow-ups to the client? Elena collects, Renata proposes, but no agent owns ongoing client communication.
→ Solution: Add Rosa — Client Communications agent to the Entrega pod. Rosa owns all outbound client contact: intake confirmation (after Elena), status updates at each phase gate, proposal delivery (after Renata), revision acknowledgements, and project milestone updates. She is active from first qualified lead through project completion. Marcela reviews Rosa's messages before they send — Rosa drafts, Marcela approves.
3
Proposal revision loop undefined — If client negotiates or rejects the proposal, who manages revisions? Who tracks client feedback?
→ Solution: Rosa receives client feedback and classifies it: scope objection (→ Tomás), budget objection (→ Bruno), or contract clause (→ Legal). The relevant agent revises, Renata reassembles, Legal re-reviews, Architect Gate re-approves. Vera tracks revision count — if more than 2 full cycles occur, she flags to COO and Marcela. Maximum 3 revision cycles before Marcela steps in directly.
4
Architect input missing in scope phase — Tomás writes scope and Bruno prices it before architect review. Risk of scoping work the architects can't or won't deliver.
→ Solution: Add a lightweight Architect SOW Review gate between Tomás and Bruno. Architects receive an Asana task with Tomás's scope document and two options: Approve or Flag with notes. This is not a full review — 24–48h turnaround expected. Bruno only prices an architect-approved scope. This gate prevents the most common source of proposal rework.
5
Payment milestones not defined — CFO needs an agreed payment schedule before Pablo builds the timeline. No agent currently owns the milestone-to-invoice mapping.
→ Solution: Bruno owns a payment schedule template as part of the budget deliverable — not just total cost, but when each payment is due (% at signing, % at design completion, % at construction start, % at delivery). CFO reviews and approves the schedule. Controller uses it throughout the project to issue invoices automatically when Vera logs each milestone. No manual invoicing decisions needed mid-project.
6
Architect interface with Asana undefined — Architects are human. How do they receive tasks, approve deliverables, and interact with the system during active projects?
→ Solution: Architects interact via Asana email notifications only — no requirement to use the Asana UI actively. When their input is needed (SOW review, proposal approval, active project decisions), they receive a formatted email with the document and a single-action reply (Approve / Flag with notes). For Phase 1 this is sufficient. When humans join the workspace fully, architects get Asana seats and can participate directly. Vera owns the task of keeping architect-facing tasks clear, concise, and infrequent — no notification overload.
7
Marketing trigger timing undefined — Does Arquitecto fire at project completion only, or in parallel during active projects?
→ Solution: Two formal trigger types: (a) In-process trigger — Marcela manually signals Arquitecto when architects have enough material (site documentation, process photos) regardless of completion status. This enables Habitar-series in-process campaigns like Casa Horizonte. (b) Completion trigger — Vera automatically signals Arquitecto when a project closes, for final campaign with full asset library. Both coexist. Arquitecto checks which trigger type fired and adjusts the brief accordingly — in-process campaigns use process documentation; completion campaigns use final photography.
