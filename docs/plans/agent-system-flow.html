Document Type Reference
Inputs — consumed by agents
Documents that exist before a phase begins. Always present in Asana as task attachments or file system references.
Outputs — produced by agents
New documents created during a phase. Become inputs for the next phase or are delivered to clients/architects.
Archived — stored, not forwarded
Intermediate versions, drafts, raw materials. Kept for reference and audit trail but not passed downstream.
YAML Handoff Schema
All agent-to-agent handoffs use structured YAML with: project_id, from_agent, to_agent, current_phase, status, required_inputs, outputs_created, decisions_needed, blockers, next_action, timestamp
