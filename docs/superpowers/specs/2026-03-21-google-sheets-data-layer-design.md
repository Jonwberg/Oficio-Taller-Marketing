# Google Sheets Data Layer — Design Spec
**Date:** 2026-03-21
**Project:** Oficio Taller — Studio Data Layer

---

## Goal

Replace hand-crafted JSON files with a Google Sheets workbook as the canonical source of truth for all studio data. Agents extract data from source documents and write rows to Sheets. A sync script pulls from Sheets, runs cascade calculations, and regenerates the JSON files that the existing dashboards already consume — with no changes to any dashboard.

---

## Architecture

```
Source documents (scope PDFs, invoices, bank statements, photos)
    ↓  agents extract → write rows via sheets_client.py
Google Sheets workbook: "Oficio Taller — Studio Data"
  ├── Projects      (master record per project)
  ├── Milestones    (payment schedule rows)
  ├── Leads         (inquiries and conversion tracking)
  ├── Campaigns     (campaign briefs, platforms, metrics)
  ├── Invoices      (billing records)
  ├── Bank          (bank statement rows, reconciliation)
  └── Assets        (photo inventory, Drive links)
         ↓  scripts/sheets_sync.py (on demand)
data/project-timelines.json    ← same format, regenerated by sync
data/financial-model.json      ← same format, regenerated by sync
         ↓
Studio OS dashboards (zero changes required)
```

**Credentials:** Google Cloud service account. Key stored as `credentials.json` at project root (gitignored). Spreadsheet ID stored in `.env` as `GOOGLE_SHEETS_ID`. `SheetsClient` reads `GOOGLE_SHEETS_ID` from environment and locates `credentials.json` at a path given by the `GOOGLE_CREDENTIALS_PATH` env var (default: `credentials.json` relative to the project root).

## Workbook Schema

**Convention:** Columns marked *(sync-written)* are populated by `sheets_sync.py` only — they are present in the sheet for human visibility but must not be manually edited. All other columns are human-editable inputs.

### Projects
One row per project. Raw inputs only; sync-written columns are computed and backfilled by `sheets_sync.py`.

| Column | Type | Notes |
|---|---|---|
| id | string | e.g. `73-barra` — primary key |
| number | string | Short project number e.g. `"35"`, `"07"` |
| name | string | Project display name |
| type | string | `residential` \| `commercial` |
| tier | string | `small_residential` \| `mid_residential` \| `large_residential` \| `commercial` |
| area_m2 | number | Gross area in square metres |
| cost_per_m2 | number | Estimated construction cost per m² (MXN) |
| total_budget_mxn | number | *(sync-written)* `area_m2 × cost_per_m2` |
| fee_pct | number | Architecture fee % (default: 12) |
| estimated_fee_mxn | number | *(sync-written)* `total_budget_mxn × fee_pct / 100` |
| scope_year | number | Year scope was signed |
| scope_signed_date | string | `YYYY-MM-DD` — anchor for milestone date derivation |
| delivery_year | number | Year delivered (empty if in-process) |
| current_phase | string | `scope_signed` \| `conceptual` \| `anteproyecto` \| `ejecutivo` \| `complete` |
| status | string | `built` \| `in_process` |
| fy27 | boolean | TRUE if FY27 marketing priority |
| drive_id | string | Google Drive project folder ID |
| last_known_activity | string | `YYYY-MM-DD` — most recent activity date (in-process projects) |
| notes | string | Free text |

## Cascade Calculations

All calculations run in `sheets_sync.py`. Sheets stores raw inputs only — sync-written columns are computed and written back to Sheets for human visibility, then emitted to JSON.

```
area_m2 × cost_per_m2                        → total_budget_mxn
total_budget_mxn × fee_pct / 100             → estimated_fee_mxn
estimated_fee_mxn × [0.30, 0.20, 0.20, 0.30] → milestone amount_mxn (M1–M4)
current_phase → milestone status              → "collected" if trigger ≤ current_phase, else "projected"
```

**Milestone date derivation** (used only when `estimated_date` is blank in the Milestones sheet):

Anchor: `scope_signed_date` from the Projects sheet (full `YYYY-MM-DD`). If `scope_signed_date` is blank, fall back to `{scope_year}-01-01`. Durations are **calendar weeks**:

| Milestone | Duration from anchor |
|---|---|
| M1 (Advance / scope_signed) | 0 weeks — same month as `scope_signed_date` |
| M2 (Conceptual) | +10 weeks |
| M3 (Anteproyecto) | +10 + 10 = +20 weeks |
| M4 (Ejecutivo) | +20 + 12 = +32 weeks |

Output format: `"YYYY-MM"` (truncate to month).

**Milestone status rule** (phase sequence: `scope_signed → conceptual → anteproyecto → ejecutivo → complete`):
- Milestones whose `trigger` is at or before `current_phase` in the sequence → `"collected"`
- Remaining milestones → `"projected"`
- All milestones on `status = "built"` projects → `"collected"`

## Output JSON Format

`sheets_sync.py` must regenerate both files in their **exact existing formats**. The dashboards read these formats directly and must not be changed.

### `data/financial-model.json` structure

```json
{
  "_schema": "financial-model-v1",
  "_last_updated": "YYYY-MM-DD",
  "_description": "...",
  "_data_disclaimer": "All figures are estimates derived from fee formulas, not accounting records.",
  "assumptions": {
    "architecture_fee_pct": 12,
    "construction_cost_per_sqm_mxn": { ... },
    "payment_schedule_pct": [30, 20, 20, 30],
    "payment_schedule_labels": ["Advance", "Conceptual", "Anteproyecto", "Ejecutivo"],
    "phase_duration_weeks": { "estudios_previos": 3, "conceptual_anteproyecto": 10, "ejecutivo": 12 },
    "tier_defaults": { ... }
  },
  "projects": [ ... ]
}
```

The `assumptions` block and all top-level metadata keys (`_schema`, `_last_updated`, `_description`, `_data_disclaimer`) are **static** — they are hard-coded in `sheets_sync.py` and not stored in Sheets. The sync script sets `_last_updated` to today's date on each run.

Per-project keys in `financial-model.json` use these exact names:
- `estimated_fee_mxn` (not `estimated_fee`)
- `amount_mxn` on milestone objects (not `amount`)

### `data/project-timelines.json` structure

Contains per-project `phases` (nested object with `scope_signed`, `delivery`, `construction_end`, `press` — each with `date`, `confidence`, `note`), `drive_id`, `number`, `last_known_activity`, and `durations`. The sync script sources these from the Projects sheet columns (`drive_id`, `number`, `last_known_activity`, `scope_signed_date`) and constructs the `phases` object from them. The `confidence` and `note` sub-fields default to `"medium"` and `""` respectively unless overridden in the notes column.

## `SheetsClient` Interface

`scripts/sheets_client.py` exposes both a Python API (for agent imports) and a CLI entry point (for Bash-based agents), following the same pattern as `entrega/asana_client.py`.

**Python API:**

```python
from scripts.sheets_client import SheetsClient

client = SheetsClient()

# Read all rows from a sheet as list of dicts
rows = client.read_sheet("Projects")

# Append a new row (dict keys must match sheet column headers)
client.append_row("Leads", {"id": "L-042", "date": "2026-03-21", ...})

# Update an existing row by matching on a key column (default: "id")
client.update_row("Invoices", "INV-018", {"status": "paid", "payment_date": "2026-03-20"})

# Upsert: append if id not found, update if found
client.upsert_row("Projects", row_dict, key="id")
```

**CLI entry point:**

```bash
# Read
python scripts/sheets_client.py read Projects

# Append
python scripts/sheets_client.py append Leads --id L-042 --date 2026-03-21 --client_name "Ana García"

# Update
python scripts/sheets_client.py update Invoices INV-018 --status paid --payment_date 2026-03-20
```

## Google Cloud Setup (manual steps, documented in `docs/setup/google-sheets-setup.md`)

Steps must be followed in this exact order:

1. Go to [console.cloud.google.com](https://console.cloud.google.com), create a new project named "oficio-taller-studio"
2. Enable the **Google Sheets API** and **Google Drive API** for that project
3. Go to IAM → Service Accounts → Create service account named "studio-sync"
4. Create a JSON key for that service account → download as `credentials.json` → place in project root
5. Note the service account email (format: `studio-sync@oficio-taller-studio.iam.gserviceaccount.com`)
6. In Google Sheets, create a new blank spreadsheet named **"Oficio Taller — Studio Data"**
7. Share the spreadsheet with the service account email, granting **Editor** access
8. Copy the spreadsheet ID from the URL (the long string between `/d/` and `/edit`) into `.env` as `GOOGLE_SHEETS_ID=<id>`
9. Run `python scripts/setup_sheets.py` to create all tabs with headers and seed from existing data
10. Run `python scripts/sheets_sync.py` to verify sync produces identical JSON output

## Migration

`scripts/setup_sheets.py` seeds the workbook with all 34 existing projects by reading **both** `data/financial-model.json` and `data/project-timelines.json`, merging them by project `id`. Fields sourced per file:

- From `financial-model.json`: `id`, `name`, `type`, `tier`, `estimated_fee_mxn`, `fee_pct`, `scope_year`, `delivery_year`, `current_phase`, `status`, `fy27`, and milestone rows
- From `project-timelines.json`: `number`, `drive_id`, `last_known_activity`, `scope_signed_date` (from `phases.scope_signed.date`)

After running `setup_sheets.py`, run `sheets_sync.py` and confirm output JSON is byte-for-byte equivalent to the originals (modulo `_last_updated`).

## Out of Scope

- No changes to any dashboard HTML file
- No database (SQLite, Postgres) — Sheets is the only store
- No real-time sync — sync is on-demand via CLI command
- No authentication UI — service account only
- No formula columns in Sheets — all calculations in Python
